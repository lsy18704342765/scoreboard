# 🏆 计分神器 - 微信小程序项目

<p align="center">
  <img src="assets/tab/home-active.png" alt="Logo" width="80" height="80" style="border-radius: 20%;" />
</p>

`计分神器` 是一款专为**台球（中式八球/九球）**和**麻将**等多人游戏设计的微信小程序，提供极致丝滑的实时计分、房间结算、战绩统计及排行榜功能。项目基于**微信原生小程序框架**与**微信云开发（Cloud Development）**构建，实现了全栈无服务器架构。

项目创新性地研发了**完全离线、低延迟的语音播报系统**，无需任何外部语音插件或后端合成服务，通过极致的本地单例音频播放器与优先级队列，为玩家提供“即刻给分、即刻播报”的沉浸式运动娱乐体验。

------

## ✨ 核心特性

- 🎙️ **完全本地离线语音播报**
  - **单实例播放控制**：采用 `InnerAudioContext` 单例播放器配合优先级插队队列，有效解决多玩家快速计分时声音重叠、断音或卡顿问题。
  - **整句识别与兜底切片**：优先尝试匹配特定游戏术语（如台球的“大金”、“小金”、“普胜”、“犯规”等）；若无整句匹配，则通过高精度数字转换算法，动态拆分为中文拼音切片序列进行流式播放，延迟接近 0ms。
  - **完全免费与纯离线**：不依赖微信同声传译等付费插件或第三方 TTS 接口，零成本、无网也能播。
- ⚡ **首屏加载极速优化**
  - **延迟云初始化**：在 `app.js` 启动生命周期中，采用 `setTimeout(..., 100)` 惰性初始化云开发环境，腾出主线程优先渲染首屏页面，大幅提升小程序启动速度。
  - **本地缓存加速**：用户信息及核心配置优先利用本地强缓存（LocalStorage）同步恢复，实现零网络等待的瞬间呈现。
- ☁️ **云开发全栈架构**
  - **免运维后端**：全部数据库操作与业务逻辑通过云函数承载，数据库采用高并发、弹性伸缩的云数据库（NoSQL JSON DB）。
  - **原子化计分**：针对多玩家并发计分，设计了原子性云操作接口，保证数据的一致性与准确性。
- 🎨 **极简高质感视觉设计**
  - 符合现代审美的冷淡高质感主题，淡灰色背景辅以极具生机的抹茶绿（`#56b87e`）高亮色。
  - 响应式多玩家网格布局，适配各类全面屏手机，交互触感灵敏。

## 🗄️ 数据库设计 (WeChat Cloud Database)

小程序在云开发中使用了 **5 个核心 JSON 数据库集合**（参见 `cloudfunctions/db_init.json`）：

1. **`app_users` (用户集合)**
   - 存储用户的微信基础资料及自定义属性。
   - 索引：`wechatOpenId`（唯一索引）、`_id`。
2. **`game_rooms` (房间集合)**
   - 记录房间的状态（等待中、进行中、已结算）、房主、游戏类型及模式。
   - 索引：`roomCode`（房间号，唯一索引）、`ownerUserId`、`status`、`createdAt`。
3. **`score_records` (计分明细集合)**
   - 记录每一笔计分或扣分操作（支持多玩家原子级批量给分，支持带备注动作）。
   - 索引：`roomId`、`operatorUserId`、`targetUserId`、`createdAt`。
4. **`settlements` (结算结果集合)**
   - 记录房间对局结算时的总积分情况及结算人。
   - 索引：`roomId`（唯一索引）、`settledByUserId`。
5. **`player_results` (玩家战绩聚合集合)**
   - 归档每个玩家在每场对局中的胜负结果与积分变动，便于排行榜及个人主页统计。
   - 索引：`userId`、`roomId`、`gameType`。

## 🎙️ 语音播报切片说明

离线语音服务 `utils/voice.js` 依赖 `assets/voice/` 下的本地音频切片。若需正常播报，请确保该目录下存在以下音频文件（`.mp3` 格式）：

| 音频文件名                    | 播报场景/对应中文 | 备注              |
| :---------------------------- | :---------------- | :---------------- |
| `received.mp3`                | "收到"            | 用于分数播报前缀  |
| `points.mp3`                  | "分"              | 用于分数播报后缀  |
| `digit_0.mp3` ~ `digit_9.mp3` | "零" 到 "九"      | 数字基础切片      |
| `unit_10.mp3`                 | "十"              | 计数单位          |
| `unit_100.mp3`                | "百"              | 计数单位          |
| `unit_1000.mp3`               | "千"              | 计数单位          |
| `unit_10000.mp3`              | "万"              | 计数单位          |
| `action_normal.mp3`           | "普胜"            | 台球专属术语      |
| `action_small_gold.mp3`       | "小金"            | 台球专属术语      |
| `action_big_gold.mp3`         | "大金"            | 台球专属术语      |
| `action_foul.mp3`             | "犯规"            | 通用/台球专属术语 |
| `switch_ok.mp3`               | "切换成功"        | 系统语音反馈      |

### 🔊 播报合成逻辑示例：

- 若玩家被给分 `125` 分：
  - 系统将其拆解为 tokens: `['received', 'digit_1', 'unit_100', 'digit_2', 'unit_10', 'digit_5', 'points']`
  - 顺序播放：*“收到” -> “一” -> “百” -> “二” -> “十” -> “五” -> “分”*。
  - 片段间隔（`CLIP_GAP_MS`）设为 `25ms`，保证发音流畅连贯。
- 若触发特定台球动作 “大金”：
  - 直接播放 `action_big_gold.mp3` (*“大金”*)。

## 🚀 部署与运行指南

### 前期准备

1. 安装最新版 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
2. 准备一个已开通 **微信云开发** 权限的小程序账号（获取小程序 `AppID`）。

### 第一步：导入项目与配置

1. 打开微信开发者工具，选择 **导入项目**。

2. 目录选择 `frontend-wxapp`，填入你的小程序 `AppID`。

3. 打开 `utils/cloud-config.js`，将 `CLOUD_ENV` 替换为你在微信云开发控制台创建的**环境 ID**：

   ```javascript
   module.exports = {
     CLOUD_ENV: '你的微信云开发环境ID', 
     COLLECTIONS: {
       USERS: 'app_users',
       ROOMS: 'game_rooms',
       // ...
     }
   };
   ```

### 第二步：部署云函数

1. 在微信开发者工具左侧目录树中，右键点击 `cloudfunctions` 文件夹。
2. 选择 **“新建并云部署：所有云函数”**，或者依次右键点击以下云函数并选择 **“下载依赖并云部署”**：
   - `user-login`
   - `room-crud`
   - `room-score`
   - `stats-query`

### 第三步：初始化云数据库

1. 在开发者工具上方点击 **“云开发”** 按钮打开云控制台。
2. 进入 **“数据库”** 标签页。
3. 根据 `cloudfunctions/db_init.json` 的指引，**新建以下 5 个集合**（保持名称一致）：
   - `app_users`
   - `game_rooms`
   - `score_records`
   - `settlements`
   - `player_results`
4. 为各集合建立对应的索引（例如，为 `game_rooms` 集合的 `roomCode` 字段建立 `Unique` 唯一索引），以提升高并发时的查询及去重效率。

## 🛠️ 技术选型与规范

- **前端技术**：微信原生小程序语法（WXML, WXSS, JS, JSON） + ES6 标准。
- **底层通信**：摒弃传统 HTTPS 请求，全量采用小程序原生的 `wx.cloud.callFunction`，减少网络握手，数据交互格式原生支持 JSON，具备极佳的安全性。
- **设计标准**：
  - 遵守微信小程序官方设计指南。
  - 所有全局通用色度及字体尺寸均提取在 `app.wxss` 中的 `:root` 变量中，方便后续进行全局换肤与暗黑模式适配。
